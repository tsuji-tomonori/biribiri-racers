# .workspace デザイン差分修正

状態: in_progress

## 背景

ユーザーから、現在の Web UI が `/home/t-tsuji/project/biribiri-racers/.workspace` の提示画像と「全く似ていない」ため、差分を明確にしたうえで同じデザインへ寄せるよう依頼された。

## 目的

既存の静的 Web アプリを、参照画像の「かわいい、ポップ、電撃、玩具風、アーケードUI」へ近づける。特に、Webカード風に見える構造、簡易CSSコース、薄い装飾密度を改善する。

## タスク種別

修正

## 軽量なぜなぜ分析

### 問題文

2026-05-16 時点の `app/web` は、仕様書の文言や画面は一部実装済みだが、`.workspace` の参照画像と比べて、画面比率、コース表現、UI密度、ぷっくりした質感、電撃演出が不足し、同じデザインに見えない。

### 確認済み事実

- 参照画像は 16:9 UI 画像と 1:1 コース画像で構成されている。
- 既存実装は `app/web/index.html`、`app/web/styles.css`、`app/web/app.js` の静的 Web 構成である。
- 既存 HOME は左右2カラムの Web カード構成で、参照画像の 1920x1080 ゲーム画面風レイアウトと異なる。
- 既存コース表現は CSS / canvas の簡易描画で、参照画像の高密度なブロック壁、電撃、玩具風コース画像と異なる。
- `app/web/app.js` には `courseId` ベースの `courses` マスタがあり、ここは継続利用できる。

### 推定原因

- 仕様文言と主要画面の追加を優先し、参照画像そのものの比率・アセット密度・視覚階層を再現する共通UI基盤が不足した。
- コース画像を静的アセットとして取り込まず、簡易CSS/canvas表現で代替していたため、最も目立つビジュアル差分が残った。
- 画面レイアウトが通常のレスポンシブ Web grid 中心で、1920x1080 のゲームUI座標に近い配置になっていなかった。

### 根本原因

`.workspace` の実画像を基準にしたデザイン基盤がなく、UI 実装が「仕様項目を満たす Web ページ」に留まり、「参照画像と同じゲームUI」に寄せる工程が不足していた。

### 対策方針

- 1:1 コース参照画像を Web アセットとして取り込み、カード・詳細・HOME・ゲーム背景の主要ビジュアルに使う。
- HOME / TEAM_CREATE / MAP_LIST / GAMEPLAY / RESULT のレイアウトを 16:9 のゲームUI密度に寄せる。
- 共通のパネル、ボタン、カード、ルームコード、ヘッダーを白フチ・厚い影・内側ハイライト・発光へ強化する。
- 未接続のオンライン機能は実データ風に見せず、明示的な未接続状態として維持する。

## スコープ

- 対象: `app/web` の UI / CSS / JS / README、task/report
- 対象外: オンライン対戦実装、コード検索サービス、永続ランキング、collision mask 分離

## 実装計画

1. コース参照画像を WebP 化して `app/web/assets/courses/` に取り込む。
2. `courses` マスタに `previewAsset` / `mapAsset` を追加する。
3. HOME と各画面の CSS を 16:9 ゲーム画面風に調整する。
4. `mini-map-art` / canvas 背景 / コースカードで参照画像を使う。
5. README に参照画像アセット方針を追記する。
6. 検証と作業レポートを実施する。

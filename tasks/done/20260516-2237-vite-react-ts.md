# Vite React TypeScript 移行

状態: done

## 背景

`app/web` は静的 HTML/CSS/JavaScript として実装されている。今後の画面追加、状態管理、ゲーム UI の保守性を上げるため、Vite + React + TypeScript 構成へ移行し、UI を責務単位のコンポーネントへ分割する。

## 目的

- `app/web` を Vite + React + TypeScript で起動・ビルドできる構成にする。
- 既存の主要 UI とローカル一人プレイ動線を維持する。
- データ、共通 UI、画面、canvas ゲーム処理を分離し、コンポーネントの原子性を担保する。

## スコープ

- 対象: `app/web` のビルド設定、エントリポイント、React/TypeScript 実装、README 更新。
- 対象外: オンライン対戦、サーバー API、collision mask / tile map の新規実装、アセット追加。

## タスク種別

機能追加

## 実装計画

1. `package.json`、Vite、TypeScript 設定を追加する。
2. `index.html` を Vite の React エントリへ更新する。
3. 既存コースマスタを TypeScript データとして分離する。
4. UI primitives、feature components、screen components、hooks に分ける。
5. Canvas レース処理を React hook / component の責務へ閉じ込める。
6. README の起動・検証手順を更新する。

## ドキュメント保守計画

- `app/web/README.md` の起動方法、構成、確認コマンドを Vite + React + TypeScript 前提に更新する。
- `docs/` は実装仕様や要件の恒久変更ではなく開発構成変更のため、必要な場合のみ追加更新する。

## 受け入れ条件

- `app/web` に Vite + React + TypeScript の設定と npm scripts が存在する。
- 既存のメニュー、チーム作成、コード参加、マップ一覧、レース待機、ゲーム中 HUD、リザルトの主要画面が React コンポーネントとして表示される。
- コースデータ、共通 UI、画面、ゲーム canvas 処理が独立したファイル・責務へ分離されている。
- 本番 UI に架空の外部接続結果や実装済みでないオンライン機能を実データのように見せる fallback を追加しない。
- `npm run build` が pass する。
- `git diff --check` が pass する。

## 検証計画

- `npm install` または lockfile 作成に必要な依存解決。
- `npm run build`
- `git diff --check`
- 可能なら Vite dev server または preview を起動し、HTTP 応答を確認する。

## PR レビュー観点

- コンポーネント分割が責務単位になっているか。
- 既存 UI 表示・操作の主要動線を落としていないか。
- Vite の asset path と production build の asset 解決が正しいか。
- 未実装オンライン機能を実データとして誤表示していないか。

## リスク

- 大きな単一 JS から React へ移行するため、細部の DOM 表示や操作に差分が出る可能性がある。
- 依存解決にネットワークが必要な場合、sandbox 制約で検証が blocked になる可能性がある。

## 完了結果

- `app/web` を Vite + React + TypeScript 構成へ移行した。
- UI を `components/`、状態制御を `hooks/`、canvas / track 処理を `game/`、コース・アセット定義を `data/` に分離した。
- 静的アセットを Vite の public 配信向けに `app/web/public/assets/` へ移動した。
- `app/web/README.md` と `docs/ui-spec-asset-pack-v2-ui.md` を新構成に合わせて更新した。
- PR #8 を作成し、受け入れ条件確認コメントとセルフレビューコメントを追加した。

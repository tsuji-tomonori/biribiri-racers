# asset pack v2 を使った UI デザイン修正

- 状態: done
- タスク種別: 修正
- ブランチ: `codex/asset-pack-v2-ui`
- 対象: `app/web/`

## 背景

ユーザーから、`.workspace/biribiri_racers_asset_pack_v2.zip` と `.workspace` のアセットを利用して、画像のような UI になるようにデザインを修正する依頼があった。

## 目的

既存の単一ページ Web アプリを、同梱された `biribiri_racers_asset_pack_v2` のコースカード、カート、エフェクト等を使ったビジュアルへ寄せる。

## スコープ

- zip 内アセットを必要最小限で `app/web/assets/v2/` に配置する。
- メニュー、マップ/コース選択、レース準備・レース中・結果画面でアセットを表示する。
- CSS を調整し、参照画像群に近い明るいキッズ向けレーシング UI にする。
- 必要な UI spec、作業レポート、検証記録を残す。

## スコープ外

- オンラインルーム接続や保存済みランキング等の未実装機能の実装。
- zip 内の全アセット取り込み。
- 本番デプロイ。

## 軽量なぜなぜ分析

- 問題文: 現在の UI は既存 CSS / 一部旧 asset ベースの表現に留まり、今回指定された `asset_pack_v2` の見た目や画像資産が十分に反映されていない。
- 確認済み事実:
  - `app/web/` は HTML/CSS/JS の静的アプリである。
  - zip には `course_cards`、`kart_sprites`、`effects_badges`、`icons` 等の PNG が含まれる。
  - 既存 JS は course の `previewAsset` / `mapAsset` を参照している。
- 推定原因:
  - 既存画面は先行 UI 仕様に合わせた CSS 図形・既存 WebP asset を中心にしており、v2 zip への asset mapping が未実施。
  - 表示コンポーネントの画像参照が v2 の命名・配置に接続されていない。
- 未確認点:
  - ユーザーの「画像」が `.workspace` 内のどの PNG を指すかは明示されていないため、同梱 asset pack と既存参照画像群から推定する。
- 根本原因:
  - 新 asset pack を UI 仕様・asset mapping・実装へ接続する工程がまだ反映されていない。
- 対策:
  - UI spec に対象参照と asset mapping を記録する。
  - v2 asset を静的配信パスへ配置し、JS の course metadata と CSS の表示に接続する。
  - ブラウザ QA で desktop / mobile の表示崩れと asset 読み込みを確認する。

## 実装計画

1. zip と `.workspace` の参照画像を確認し、UI spec と asset mapping を作成する。
2. 必要アセットのみ `app/web/assets/v2/` へ展開する。
3. `app/web/app.js` の course / sprite / effect 参照を v2 asset へ切り替える。
4. `app/web/index.html` と `app/web/styles.css` を調整し、画像主体の UI にする。
5. desktop 1440px と mobile 375px でブラウザ確認し、必要に応じて修正する。
6. 検証、作業レポート、commit / PR / PR コメントまで進める。

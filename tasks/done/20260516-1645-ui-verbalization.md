# UI 言語化ドキュメント追加

- 状態: done
- タスク種別: ドキュメント更新
- 作成日時: 2026-05-16 16:45 JST

## 背景

ユーザーから「.workspace できる限りUIを言語化して」と依頼された。リポジトリの Web UI 実装を読み取り、画面・状態・操作・視覚表現を後続作業者が把握できる文章として残す。

## 目的

`app/web` の UI をできる限り具体的に言語化し、実装・レビュー・改善時の参照資料にする。

## スコープ

- 対象: `app/web/index.html`、`app/web/styles.css`、`app/web/app.js` から読み取れる UI
- 成果物: UI 言語化ドキュメント、README からの導線、作業レポート
- 対象外: UI 挙動の実装変更、オンライン対戦など未実装機能の実装

## 実施計画

1. 既存 UI 実装と README を確認する。
2. 画面別に目的、構造、表示文言、操作、状態遷移、視覚表現を整理する。
3. UI 言語化ドキュメントを追加し、README からリンクする。
4. 変更範囲に応じた Markdown 検証を実施する。
5. 作業完了レポートを作成する。

## ドキュメント保守方針

今回は UI 実装そのものは変えず、既存 UI の説明を `app/web` に近接したドキュメントとして追加する。`docs/` 配下の要件書ではなく、現在の Web 初版 UI の参照資料として扱う。

## 受け入れ条件

- [x] メニュー、ルーム作成、レース開始前、ゲーム中、リザルト、モーダルの各 UI が文章で説明されている。
- [x] 画面遷移、主要操作、キーボード操作、レース状態、表示更新が説明されている。
- [x] 視覚トーン、レイアウト、色、装飾、アクセシビリティ上の既存要素が説明されている。
- [x] 未実装・準備中の UI が実データではなく将来拡張表示であることを明記している。
- [x] README から追加ドキュメントへ辿れる。
- [x] 変更範囲に見合う検証を実施し、結果または未実施理由を記録している。

## 検証計画

- `git diff --check`
- Markdown ファイルの目視確認
- `pre-commit run --files <changed-files>` は利用可能なら実施する

## PR レビュー観点

- 実装と矛盾する説明がないか。
- 未実装機能を実装済みのように書いていないか。
- UI 言語化が後続の UI 改善や QA に使える粒度になっているか。

## リスク

- 実装を読んだ静的な言語化であり、ブラウザ上の実表示確認を行わない場合は視覚再現の細部に差分が残る可能性がある。
- 既存 UI に含まれる固定表示や将来拡張表示は、実装変更ではなく説明上の明確化に留める。

# ビリビリレーサーズ Web 版

`app/web` は、ビリビリ棒のように壁へ触れるとスタートへ戻る 2D レーシングゲームの Vite + React + TypeScript 版です。
提示デザイン仕様に沿って、メニュー、チーム作成、コード参加、マップ一覧、レース待機、ゲーム中 HUD、リザルトの主要画面をローカル一人プレイ向けに実装しています。
現在の UI は asset pack v2 の PNG を主役にし、`app/web/public/assets/v2/` の course-card、course-parts、buttons、ui、logo、effects、icons、karts、themes を HOME preview、コースカード、マップ詳細、待機 / ゲーム中 HUD、リザルト周辺へ使用しています。

## 構成

- `src/App.tsx`: 画面遷移、選択コース、プレイヤー設定、レース結果のアプリ状態を管理します。
- `src/components/ui/`: ボタン、画面 wrapper、モーダルなどの共通 UI primitives です。
- `src/components/course/`: コースカード、コース詳細、コースパーツ表示を扱います。
- `src/components/screens/`: メニュー、チーム作成、コード参加、マップ、待機、ゲーム、リザルトの各画面です。
- `src/data/`: コースマスタとアセットパスを保持します。
- `src/game/`: canvas 描画、チップ構成のコース仕様、画像 preload を扱います。
- `src/game/chips/`: コース別の走行ライン、道幅、壁チップ配置、床パターン、ミニマップ縮小描画、Path2D ベースの当たり判定を管理します。
- `src/hooks/useRaceController.ts`: キー入力、カウントダウン、レースループ、HUD 更新を閉じ込めます。

## 起動方法

初回は `app/web` で依存関係をインストールします。

```bash
cd app/web
npm install
```

開発サーバー:

```bash
npm run dev -- --host 127.0.0.1
```

ブラウザで Vite が表示する URL を開きます。
画面確認用に、`?screen=room`、`?screen=join`、`?screen=map`、`?screen=ready`、`?screen=result` で主要画面を直接開けます。

## 操作

- `WASD` または `矢印キー`: カート移動
- `Space`: ブースト
- `Enter`: レース開始前画面で即スタート
- `Esc`: モーダルを閉じる、またはチーム作成 / コード参加 / マップ一覧からメニューへ戻る

## 画面

- メニュー画面: ルーム作成、コードで参加、遊び方、設定、マップ一覧への導線
- チーム作成画面: チーム名、最大人数、公開設定、任意パスワード、壁ルール、ラップ数、ブースト、コース選択
- コードで参加画面: `AAAA-9999` 形式の入力確認と、オンライン検索未接続の明示
- マップ一覧画面: `courseId` ベースのコースマスタから 6 コースを表示
- レース待機画面: コース情報、ルール、PC 操作、準備 OK 表示
- ゲーム中画面: ランキング、ミッション、ミニマップ、ブースト、PC/タッチ HUD
- リザルト画面: 優勝者、タイム、壁接触回数、今回のコース、再戦導線

## コースデータ

コース表示は `app/web/src/data/courses.ts` の `courses` マスタを正とします。
画面内の表示名は `courseId` から参照し、名称揺れは `aliases` に保持します。
各コースは `previewAsset` / `mapAsset` に v2 の course-card PNG パスを持ち、UI はこの値からサムネイルとプレビューを描画します。
course-card PNG にはコース番号、名称、星、説明、ラップ、目安時間が焼き込まれているため、HTML 側では画像上に同じラベルを重ねず、隣接詳細やアクセシブルネームにコースデータを使います。
ゲーム中 canvas とレース待機 canvas は `src/game/chips/` のコース仕様を使います。`themeAssets.floor` を床パターンとして敷き、`partAssets` を左右の壁チップとして繰り返し配置します。画像ロードに失敗した場合も、グラデーション床、グリッド、角丸矩形チップのフォールバックで描画を継続します。
当たり判定、進捗率、スタート復帰位置はコース別の中心線 `Path2D` と `roadWidth` から取得し、背景画像のピクセル色には依存しません。ミニマップも同じコース仕様を縮小描画します。

実装済み表示:

- `01`: パステルプラネット
- `02`: ピカピカシティ
- `03`: キャンディループ
- `04`: サンダーガーデン
- `05`: アイスキューブ
- `06`: スカイスパイラル（詳細マップ未確定）

各画面の構造、表示文言、操作、状態、視覚表現は [`UI_DESCRIPTION.md`](./UI_DESCRIPTION.md) にまとめています。

## 初版の制約

- オンライン対戦、コード参加の実検索、フレンド、ランキング共有は未実装です。
- ルームコード、複数人枠、共有ボタンは将来拡張のための表示であり、接続可能な実データとして扱いません。
- 記録はブラウザ内の一時状態で、ページリロードにより消えます。
- レース canvas はコース別のチップ構成と `Path2D` 判定を使います。背景画像のピクセル色 collision mask やコース画像そのものの tile map 判定は使っていません。
- `06` スカイスパイラルは詳細マップ未確定ですが、レース canvas では暫定チップコース仕様と course-parts フォールバックで描画します。

# ビリビリレーサーズ Web UI 言語化

この文書は、`app/web` の現行 UI を、画面設計・視覚表現・操作・状態の観点で言語化したものです。実装の参照元は `index.html`、`styles.css`、`app.js` です。

## 1. 全体像

ビリビリ棒の「壁に触れたらアウト」という緊張感を、明るいキッズ向けレースゲームのトーンで表現する一人プレイ Web UI です。初版はオンライン対戦ではなく、1 人でルーム設定、レース準備、走行、リザルト確認までを行います。

画面は次の 5 つです。

- メニュー画面: タイトル、主要導線、コースの雰囲気を見せる入口。
- ルーム作成画面: プレイヤー名、ラップ数、ブースト、コースを設定する画面。
- レース開始前画面: カウントダウン、コースプレビュー、操作説明を見せる画面。
- ゲーム中画面: HUD、キャンバスのレース画面、ミニマップ、ステータスを表示する画面。
- リザルト画面: 完走タイム、接触回数、暫定順位を表示する画面。

共通要素として、あそびかた・設定のモーダルがあります。画面切り替えはページ遷移ではなく、`screen` セクションの表示状態を切り替えて行います。

## 2. UI トーン

### 2.1 視覚テーマ

全体の印象は「電撃」「おもちゃのレース」「ポップなゲームセンター」です。青、水色、ピンク、黄色、ティールを強く使い、白い太枠と影でボタンやパネルを立体的に見せます。

背景は水色から青への斜めグラデーションです。白い半透明の模様、斜めストライプ、雷・星・ひし形の装飾が重なり、静止画でもにぎやかな印象になります。

### 2.2 形状

パネルやボタンは大きな角丸を持ちます。メインボタンは厚みのある影、白い外枠、押下時の沈み込みで「押せる」感覚を出しています。コースカード、プレイヤースロット、HUD、モーダルも同じく太枠と角丸で統一されています。

### 2.3 タイポグラフィ

日本語は丸ゴシック系を優先しています。見出しは太く、影付きで、ゲームタイトルや勝利表示は大きく扱います。説明文や補助テキストも太めで、子ども向けの読みやすさとゲームらしい勢いを優先しています。

## 3. 共通レイアウト

### 3.1 アプリ外枠

`app-shell` が画面全体を包み、最小高さはビューポート全体です。内側の `game-app` は最大幅 1680px に制限され、広い画面では中央に配置されます。

### 3.2 画面表示

すべての画面は `.screen` として定義されています。通常は非表示で、`.is-active` が付いた画面だけが表示されます。メニュー画面だけはアクティブ時に 2 カラムのグリッドになります。

### 3.3 レスポンシブ

1180px 以下では、メニュー、ルーム、レース準備、ゲーム中、リザルトの主要グリッドが 1 カラムに変わります。ゲーム中の左右 HUD は横並びから、複数カードが折り返すレイアウトになります。

720px 以下では余白、角丸、ボタン内アイコン、見出し、キャンバス高さが縮小されます。フッター操作ボタンやリザルト操作ボタンは縦並びになり、各ボタンが横幅いっぱいになります。

### 3.4 フォーカス

ボタン、入力欄、キャンバスには `:focus-visible` で黄色の太いアウトラインが付きます。キーボード操作時に、どの UI 部品へフォーカスしているか視認できる設計です。

### 3.5 モーション低減

`prefers-reduced-motion: reduce` に対応し、該当環境ではトランジションやアニメーションの時間をほぼゼロにします。

## 4. メニュー画面

### 4.1 目的

ゲームの入口として、タイトル、遊び始める導線、ルーム作成導線、説明・設定導線、コースの雰囲気を一画面で伝えます。

### 4.2 左側ヒーローパネル

左側には白い半透明パネルがあり、上から順に以下を表示します。

- 小さな帯文: 「さわるとビリビリ！ はしりぬけて、いちばんのりをめざそう！」
- タイトル: 「ビリビリ」「レーサーズ」を改行して大きく表示。
- メインメニュー: 4 つの大きなボタン。
- お知らせバー: 初版が一人プレイ対応で、オンライン参加は準備中であること。

タイトルは青い大文字風の太字で、白と水色の影を重ね、ロゴのように見せています。

### 4.3 メインメニューボタン

ボタンは縦に 4 つ並びます。各ボタンは左にアイコン、右に太字の主文言と小さな補助文言を持ちます。

- ピンク: 「ひとりであそぶ」。すぐにレースを始める導線。
- 青: 「ルーム作成」。一人用ルーム設定へ進む導線。
- ティール: 「あそびかた」。説明モーダルを開く導線。
- 黄色: 「設定」。設定モーダルを開く導線。

「ひとりであそぶ」と「ルーム作成」はどちらもルーム作成画面へ遷移します。初版では一人プレイ用の設定画面がレース開始前の入口を兼ねています。

### 4.4 右側ショーケースパネル

右側はゲームの雰囲気を見せる装飾的な領域です。上部にプロフィールカード、中央にコースプレビュー、下部におすすめコースカードがあります。

プロフィールカードには雷アイコンのアバター、「ビリビリくん」「一人プレイ」が表示されます。コースプレビューは CSS で描いたトラック、3 台の車、ゴールバナーを見せます。この領域は `aria-hidden="true"` の装飾も含み、操作対象ではありません。

おすすめコースは 3 枚のカードです。

- 01 ネオンストリート: 初心者向け 01:20。
- 02 パステルプラネット: カーブ多め 02:15。
- 03 サンダーサーキット: 高速 01:58。

ここはメニュー上の見せ場であり、実際のコース選択操作はルーム作成画面で行います。

## 5. ルーム作成画面

### 5.1 目的

レース前に一人用プレイ条件を整える画面です。「チーム作成」という見出しを持ちますが、初版は一人用固定です。

### 5.2 ヘッダー

左に戻るアイコンボタン、右に説明帯と見出しがあります。戻るボタンは `aria-label="メニューへ戻る"` を持ち、メニュー画面へ戻ります。

### 5.3 チーム設定パネル

左のパネルです。次の設定行があります。

- プレイヤー名: 最大 12 文字のテキスト入力。画面上のプレイヤー名、準備画面、リザルト、ランキングに反映されます。空欄時は「ビリビリくん」に戻ります。
- ラップ数: 減らす・増やすボタンと `output`。1 から 7 の範囲で変更できます。
- 公開設定: 「一人用」と表示されます。変更操作はありません。
- 壁に触れると: 「スタートへ戻る」と表示されます。変更操作はありません。
- ブースト: ON/OFF ボタン。Space キーまたは画面ボタンで使うブーストを有効・無効にします。

変更できる項目と表示のみの項目が混在します。表示のみの項目は、初版のルールを伝えるための固定状態です。

### 5.4 プレイヤー枠パネル

中央のパネルです。見出しは「プレイヤー枠 1/4」です。1P の行だけが有効で、青い小型車、プレイヤー名、「ホスト / 準備OK」を表示します。

2P、3P、4P は「募集機能は準備中」と表示されます。これは未実装機能の将来拡張枠であり、実際のオンライン参加者や待機中ユーザーを表すデータではありません。

下部にはチームコードとして「BIRI 1234」が表示されます。これもオンライン参加は今後対応予定であることを示す表示で、実際の参加コードとしては機能しません。

### 5.5 コース選択パネル

右のパネルです。横スクロール可能なコースボタンが 3 つあります。

- 01 ネオンストリート
- 02 パステルプラネット
- 03 サンダーガーデン

選択中のボタンはピンクのグラデーションで強調されます。選択に応じて、下部のコース詳細が変わります。詳細にはミニマップ風の装飾、コース名、説明文、タグが表示されます。

コースごとの説明は次の通りです。

- ネオンストリート: ネオンが輝く夜の街を、かべに気をつけて走る。タグは「直線多め」「初心者向け」「3ラップ推奨」。
- パステルプラネット: カラフルなかべが続く、リズムよく曲がるコース。タグは「カーブ多め」「集中力」「3ラップ推奨」。
- サンダーガーデン: 明るい庭園を走る、ゆるやかだが油断できないコース。タグは「広め」「復帰しやすい」「初心者向け」。

### 5.6 フッター操作

画面下に「もどる」と「レースへ」の大きなボタンがあります。「もどる」はメニュー画面へ戻ります。「レースへ」はレース開始前画面へ進み、カウントダウンを開始します。

## 6. レース開始前画面

### 6.1 目的

レース開始直前の確認画面です。カウントダウンの演出、コースプレビュー、操作方法、プレイヤー準備状態を表示します。

### 6.2 レイアウト

3 カラム構成です。

- 左: コース情報、ルール、キー操作。
- 中央: キャンバスで描画されたコースプレビューと大きなカウントダウン。
- 右: プレイヤー準備カードと即スタートボタン。

1180px 以下では縦方向の 1 カラムになります。

### 6.3 左サイドパネル

見出しは「レーススタートまで...」です。コース情報カードには選択中コース名と「壁に触れるとスタートへ戻る」を表示します。

ルール一覧は 3 つです。

- かべに触れるとアウト。
- スタートにもどって再挑戦。
- ラップを走りきるとゴール。

キー一覧は `WASD / 矢印キー` が移動、`Space` がブーストです。

### 6.4 中央ステージ

`ready-canvas` にコースが描画されます。中央には大きなカウントダウンバッジが重なります。最初は「3」、以降「2」「1」「GO!」と更新され、その後ゲーム画面へ遷移します。

下部付近には `START` ラベルが重なり、開始位置を視覚的に示します。

### 6.5 右サイドパネル

見出しは「プレイヤー 1/1」です。プレイヤーカードには青い小型車、プレイヤー名、「準備OK」、チェックマークを表示します。

「今すぐスタート」ボタンを押すとカウントダウンを待たずにゲームを開始します。キーボードでは、レース開始前画面で Enter を押しても即スタートします。

### 6.6 TIP 表示

画面下に TIP バーがあり、「かべギリギリを攻めると速いが、ぶつからないよう注意」というプレイ助言を表示します。

## 7. ゲーム中画面

### 7.1 目的

走行中の状態を表示し、キャンバス上で車を操作する画面です。HUD により、タイム、ラップ、ランキング、ミッション、操作、ミニマップ、接触回数、ブースト状態を把握できます。

### 7.2 上部 HUD

上部にはカード型の HUD が並びます。

- ロゴミニ: 「ビリビリ」「レーサーズ」。
- タイム: 走行開始からの経過時間。`00:00.00` 形式。
- ラップ: 現在ラップと総ラップ。例: `1/3`。

1180px 以下ではロゴミニが非表示になり、タイムとラップが残ります。

### 7.3 左 HUD

左側には 3 つの HUD パネルがあります。

- ランキング: 1 位として現在のプレイヤー名を表示し、進行率をパーセントで表示します。
- ミッション: 壁に触れるとスタートへ戻ること、指定ラップ数を走ることを表示します。
- そうさほうほう: `WASD`、矢印キー、Space ブーストを表示します。

ランキングは一人プレイの走行中状態を表すもので、オンライン順位ではありません。

### 7.4 中央レースキャンバス

`game-canvas` がレースの主画面です。キャンバスには次の要素が描かれます。

- 水色系の背景と白いタイル状の模様。
- 星の装飾。
- 太い白・ピンク・ティール・灰色のトラック。
- 破線の中央線。
- START のチェッカーライン。
- ゴールバナー。
- コース上の障害物。
- 電撃のような白い線。
- 進行方向を示す黄色い矢印。
- プレイヤーの車。
- プレビュー時のみ、複数色の車。

車は青いボディ、白い縁取り、窓、ライトを持ちます。ブースト中は後方に黄色い炎のような形が描かれます。接触直後の無敵時間には車が半透明になります。

キャンバスは `tabindex="0"` を持ち、レース開始時にフォーカスされます。スクリーンリーダー向けに「ビリビリレーサーズのレース画面」という `aria-label` があります。

### 7.5 トースト

キャンバス上にはトースト領域があります。通常は透明で非表示に近い状態です。

壁に触れると「かべにせっしょく！ スタートへもどる」と表示されます。ラップ更新時は `LAP 2/3` のような表示に変わります。トーストは白いカード、ピンクの文字、太い枠、影で目立つように表示されます。

### 7.6 右 HUD

右側にはミニマップ、ステータス、ブーストボタンがあります。

ミニマップは小さなキャンバスで、コース線と現在位置を示します。ステータスには「かべ接触」と「ブースト」があり、接触回数は `0回` から増えます。ブーストは `OK`、`充電中`、`OFF` のいずれかです。

ブーストボタンは大きな円形ボタンです。雷アイコンと「つかう」を表示し、タッチ操作でもブーストできます。`aria-label="ブーストを使う"` を持ちます。

### 7.7 操作モデル

移動は `WASD` または矢印キーです。押しているキーの方向へ加速し、キーを離すと減速します。Space またはブーストボタンで短時間加速します。

ブーストは有効設定が ON のときだけ使えます。使用後は一定時間クールダウンし、HUD のブースト表示が `充電中` になります。

### 7.8 接触と復帰

車が道路外へ出ると、スタート地点へ戻ります。接触回数が増え、短い無敵時間が付きます。無敵時間中は再接触判定を受けず、車は半透明に表示されます。

### 7.9 ラップとゴール

コース上の進行率を計算し、終盤から始点近くへ戻ったときにラップ完了として扱います。設定したラップ数に達するとリザルト画面へ遷移します。

## 8. リザルト画面

### 8.1 目的

完走後に、勝利演出、タイム、接触回数、今回のコース、順位風の結果を表示します。

### 8.2 勝利カード

左側の大きなカードに、次の要素を表示します。

- 英字見出し: `WINNER!`
- 大見出し: `ゆうしょう!`
- リボン文言: `レース おつかれさま！`
- 金メダル風の `1st`
- 大きな青い車
- プレイヤー名
- タイム
- かべに当たった回数

背景は白、黄色、ピンクを混ぜた祝祭的なグラデーションで、枠内にも黄色の線を入れています。

### 8.3 スコアパネル

右側には 2 つのパネルがあります。

1 つ目は今回のコースです。小さなミニマップ風アート、コース名、「一人プレイの記録です。オンライン順位は今後対応予定。」という説明を表示します。

2 つ目はリザルト一覧です。1st はプレイヤー名とタイムを表示します。2nd と 3rd は「オンライン対戦は準備中」と表示し、タイムは `-` です。これは未実装のオンライン対戦結果ではなく、将来拡張を示す空き枠です。

### 8.4 リザルト操作

画面下には 3 つの大きなボタンがあります。

- もういちど: 同じ設定でレース開始前画面へ戻り、カウントダウンを始めます。
- ルームへ: ルーム作成画面へ戻ります。
- メニューへ: メニュー画面へ戻ります。

## 9. モーダル

### 9.1 表示条件

メニュー画面の「あそびかた」または「設定」ボタンから開きます。どちらも同じモーダルを使い、タイトルだけが押したボタンに応じて変わります。

### 9.2 構造

画面全体に青い半透明のオーバーレイがかかり、中央に白いモーダルカードが表示されます。右上にはピンクの丸みのある閉じるボタンがあります。

モーダルは `role="dialog"`、`aria-modal="true"`、`aria-labelledby="modal-title"` を持ちます。開くと閉じるボタンへフォーカスします。Esc キーまたは閉じるボタンで閉じられます。

### 9.3 本文

本文は、矢印キーまたは WASD でカートを動かすこと、電撃の壁に触れるとスタートへ戻って接触回数が増えること、Space で短くブーストできること、規定ラップを走りきるとリザルトへ進むことを説明します。

## 10. 状態とデータ表示

### 10.1 保持する状態

画面状態、プレイヤー名、ラップ数、ブースト有効状態、選択コース、カウントダウン、レース状態、最後の結果を JavaScript のメモリ上に保持します。リロード後に保存される永続データはありません。

### 10.2 プレイヤー名

初期値は「ビリビリくん」です。入力欄が空になると同じ既定名に戻ります。表示先はプレイヤー枠、開始前画面、ゲーム中ランキング、リザルト、リザルト一覧です。

### 10.3 ラップ数

初期値は 3 です。1 から 7 まで変更できます。ルーム作成画面の出力、ゲーム中のミッション、上部 HUD、ラップ判定に反映されます。

### 10.4 コース

初期選択はネオンストリートです。コース選択は表示文言とタグを変えます。現行実装の実走行トラック形状は共通のキャンバス描画です。

### 10.5 レース結果

完走時に経過時間、接触回数、プレイヤー名、コース名を最後の結果として保持し、リザルト画面へ反映します。ランキング共有や履歴保存はありません。

## 11. アクセシビリティ上の現行要素

### 11.1 ランドマークと見出し

主要 UI は `main` 内にあります。各画面は `aria-labelledby` で画面見出しと関連付けられています。ゲーム中画面には視覚的に隠した `ゲーム中` 見出しがあります。

### 11.2 ライブ領域

`main.game-app` に `aria-live="polite"` が付いています。画面切り替えや表示更新を支援技術へ穏やかに通知する意図があります。

### 11.3 ラベル

戻るボタン、ラップ増減、キャンバス、ミニマップ、ブーストボタン、閉じるボタンなどに `aria-label` があります。装飾的なコースステージやミニマップ風アートには `aria-hidden="true"` が指定されています。

### 11.4 キーボード

ボタン、入力欄、キャンバスはキーボードフォーカス可能です。Esc でモーダルを閉じ、ルーム画面ではメニューへ戻ります。レース開始前画面で Enter を押すと即スタートします。Space はブースト操作です。

### 11.5 注意点

コース選択は `role="listbox"` を持ちますが、各選択肢は `option` ではなくボタンです。現行 UI の説明としては「ボタン型のコース選択」と扱います。将来のアクセシビリティ改善では、ボタン群としての表現または listbox パターンの完成を検討できます。

## 12. 未実装・準備中表示

現行 UI には将来拡張を示す表示があります。これらは実データではありません。

- メニューのお知らせ: オンライン参加は準備中。
- ルームの 2P、3P、4P 枠: 募集機能は準備中。
- チームコード `BIRI 1234`: オンライン参加は今後対応予定で、実際の参加コードではない。
- リザルトの 2nd、3rd: オンライン対戦は準備中。
- リザルト説明: オンライン順位は今後対応予定。

UI を変更するときは、これらを実装済み機能のように見せないことが重要です。必要なら「準備中」「今後対応予定」「一人プレイの記録」といった明示を残します。

## 13. 主な画面遷移

画面遷移は次の通りです。

- メニューの「ひとりであそぶ」または「ルーム作成」: ルーム作成画面へ。
- ルーム作成の「もどる」またはヘッダー戻る: メニュー画面へ。
- ルーム作成の「レースへ」: レース開始前画面へ進み、カウントダウン開始。
- レース開始前の「今すぐスタート」または Enter: ゲーム中画面へ。
- カウントダウン完了: ゲーム中画面へ。
- 規定ラップ完了: リザルト画面へ。
- リザルトの「もういちど」: レース開始前画面へ。
- リザルトの「ルームへ」: ルーム作成画面へ。
- リザルトの「メニューへ」: メニュー画面へ。
- メニューの「あそびかた」または「設定」: モーダル表示。
- モーダルの閉じるボタンまたは Esc: モーダル非表示。

## 14. UI 改善時の確認観点

- 準備中表示が、実際に使える機能として誤読されないか。
- プレイヤー名、ラップ数、ブースト、コース選択が全表示先に同期されるか。
- レスポンシブ時に、ボタン文言、HUD、トースト、キャンバスが重ならないか。
- キーボードだけでメニュー、ルーム設定、モーダル、開始操作、ブースト操作に到達できるか。
- タッチ操作でブーストボタンや主要ボタンのサイズが十分か。
- カウントダウン、接触トースト、ラップ更新、リザルトがユーザーに伝わるか。
- コース選択 UI の ARIA パターンが実装と一致しているか。
